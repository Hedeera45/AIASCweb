AIASC NAVIGATION BAR RESPONSIVENESS ARCHITECTURE
========================================================================

1. OVERVIEW & PROBLEM STATEMENT
------------------------------------------------------------------------
During high-DPI desktop zooming (e.g., 110% - 120%), the logical CSS width of the navigation container (.nav-bar) drops from desktop sizes down to a range of 700px - 815px. 

Under the previous design:
* All 5 buttons ("Inicio", "Publicaciones", "Organización", "Historia", "Contacto") were forced onto a single line.
* A large fixed gap of 35px, combined with 175px maximum button widths, caused the items to run out of physical space, squishing the text and causing the "Contacto" button to warp, wrap awkwardly, or collide with outer container boundaries.
* The 3-2 grid layout only triggered below 478px container width, leaving a massive gap of cramping in the medium range.

To achieve flawless, smooth, and high-performance zooming, we implemented a three-tier layout architecture.


2. PERFORMANCE CONSTRAINTS & SOLUTION
------------------------------------------------------------------------
To prevent performance degradation (layout thrashing, frame drops) on lower-end devices or high-zoom levels, dynamic layout calculations (e.g., CSS clamp(), calc()) were avoided. 

Instead, we leveraged hardware-accelerated static CSS flexbox properties. The layout now shifts dynamically across three distinct tiers based on the width of the navigation container using CSS Container Queries (@container).


3. THREE-TIER RESPONSIVE SIZING MATRIX
------------------------------------------------------------------------

TIER 1: DESKTOP LAYOUT (Container Width > 768px)
* Layout: Single-row grouped on the right.
* Gap: 35px
* Button Width Limit: flex: 0 1 175px; max-width: 175px;
* Typography: font-size: 15px; padding: 0 8px; (Premium desktop styling)
* Spacer Behavior: .nav-links::before { flex: 1 } is active, pushing the navigation flush to the right boundary.

TIER 2: COMPACT / MEDIUM ZOOM LAYOUT (Container Width: 479px - 768px)
* Layout: Single-row grouped on the right.
* Gap: 12px (Static, zero-cost gap)
* Button Width Limit: flex: 0 1 120px; max-width: 120px;
* Typography: font-size: 13px; padding: 0 6px; (High-readability, compact margins)
* Spacer Behavior: .nav-links::before { flex: 1 } is active, gracefully collapsing from 371px down to 83px to absorb shrinking width.
* Pixel Math Verification (at the absolute tightest limit of 479px):
  - "Inicio" (Text width ~35px + 12px padding)         = 47px
  - "Publicaciones" (Text width ~78px + 12px padding)  = 90px
  - "Organización" (Text width ~73px + 12px padding)   = 85px
  - "Historia" (Text width ~46px + 12px padding)       = 58px
  - "Contacto" (Text width ~56px + 12px padding)       = 68px
  - Total Buttons Width                                = 348px
  - Gaps (4 gaps * 12px spacing)                       = 48px
  - GRAND TOTAL OCCUPIED WIDTH                         = 396px
  - Safety Buffer at 479px container width: 479px - 396px = 83px (Guarantees zero wrapping or cramping)

TIER 3: MOBILE GRID LAYOUT (Container Width <= 478px)
* Layout: Multi-row, beautifully balanced 3-2 grid layout.
* Gap: 0px (flex-wrap: wrap)
* Row 1 (First 3 buttons): flex: 0 0 33.333% (Inicio, Publicaciones, Organización)
* Row 2 (Remaining 2 buttons): flex: 0 0 50% (Historia, Contacto)
* Typography: font-size: 13px; padding: 0 4px;
* Spacer Behavior: .nav-links::before { display: none } (Spacer disabled to facilitate grid flow)

5. RECENT UPDATE: VIEWPORT-BASED NAVBAR LAYOUT (6 BUTTONS)
------------------------------------------------------------------------
The container query (@container) approach described in Section 3 was abandoned because
the navbar's container width shrank independently of the viewport, causing layout fights.

Current button list (6 buttons):
  "Novedades", "Publicaciones", "Institucional", "Servicios", "Asociate", "Contacto"

The layout is now controlled by a single viewport-based breakpoint: @media (max-width: 1250px).
- Viewport > 1250px: Single row layout, all 6 buttons in one line.
- Viewport <= 1250px: 3+3 wrapped layout (flex: 0 0 33.333%), gap: 0.
  Row 1: Novedades, Publicaciones, Institucional
  Row 2: Servicios, Asociate, Contacto

The 800px breakpoint separately controls the header grid (logo + cover stacking to single column).
Both breakpoints coexist: at 800-1250px you get 3+3 nav with side-by-side header,
below 800px you get 3+3 nav with stacked header.

6. LOGO SCALING AND RESPONSIVE BEHAVIOR
------------------------------------------------------------------------
In mobile layout (@media (max-width: 800px)) or at very high zoom levels (170%+), the logo container (.logo-area) spans the full viewport width. 

Previous issue: The container had a fixed height of 90px, but the image scaled horizontally to 100% of the screen width. This inflated the logo's intrinsic height to ~200px-300px, resulting in severe bottom-cropping because the container was locked at 90px with overflow hidden.

Solution implemented:
1. Dynamic Container Height: Removed the fixed `height: 90px` on `.logo-area` for mobile and replaced it with `height: auto`. This allows the container to dynamically stretch and perfectly wrap the logo as it expands horizontally.
2. Picture Tag Constraint: Added `height: 100%` and flex centering to `.logo-area picture` to prevent a cyclic height-expansion loop that occurs when both the picture tag and the image have `height: auto`.
3. Object-Fit: `.logo-area img` maintains `width: 100%`, `height: 100%`, and `object-fit: contain` to preserve the aspect ratio safely.

7. TITLE STYLES AND TYPOGRAPHY
------------------------------------------------------------------------
The site uses a standardized set of title classes to maintain visual consistency:

A) Page Title with Green Rule (Título Principal de Página)
   - Usage: Used at the very top of pages (e.g., "CONTACTO", "QUIÉNES SOMOS").
   - Implementation: An `<h1>` with `className="page-title"` immediately followed by an `<hr>` with `className="green-rule"`.

B) Green Section Box Title (Caja Verde de Sección)
   - Usage: Used to break down content into main blocks (e.g., "HISTORIA", "PREGUNTAS FRECUENTES").
   - Implementation: Any `<h2>` placed directly inside a `<section className="section-block">`. The CSS automatically applies a green border box, uppercase text, and padding. No specific class name is needed on the `<h2>` itself.

C) Clean Subtitle (Subtítulo Limpio)
   - Usage: Used for secondary headings within sections where a full box is too heavy (e.g., "Deje un mensaje:").
   - Implementation: An `<h2>` with `className="form-subtitle"` (black text) or a custom clean subtitle class like `className="green-subtitle"` (green text, no box, no underline, matching the font size of the form subtitle).

D) Green Underlined Subtitle (Subtítulo Verde Subrayado)
   - Usage: Used for separating content items inside a section block (e.g., in Servicios or Asociate).
   - Implementation: An `<h3>` with `className="green-subtitle-underlined"`. It displays green text with a simple underline.
